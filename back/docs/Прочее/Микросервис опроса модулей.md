# Взаимодействие серверов back (алгоритм) - plc_io (опрос модулей)

## 1. back (алгоритм)

- back периодически отправляет запросы на plc_io
- back умеет работать с несколькими серверами plc_io

### Перечень запросов back - plc_io

> #### 1. rack - Отправление рамы для работы микросервиса
>
> > - Метод: `POST`
> > - url: `http://{IP:PORT}/api/back/rack`
> > - Отправляемые данные: `{ Массив ИД складов, Уникальные модули (один и тот же модуль может существовать в нескольких складах), Неисправности модулей, Количество потоков }`
> > - Получаемые данные: `пусто`
> > - Периодичность: `5 мин`
>
> #### 2. reset - Команда сброса аварии
>
> > - Метод: `POST`
> > - url: `http://{IP:PORT}/api/back/reset`
> > - Отправляемые данные: `{ Количество потоков }`
> > - Получаемые данные: `{ Обновленные показания с модулей, Неисправности модулей }`
> > - Периодичность: `Действие пользователя`
>
> #### 3. value - Запрос показания модулей
>
> > - Метод: `POST`
> > - url: `http://{IP:PORT}/api/back/value`
> > - Отправляемые данные: `params: { Количество потоков }`
> > - Получаемые данные: `{ Обновленные показания с модулей, Неисправности модулей }`
> > - Периодичность: `10 сек`
>
> #### 4. write - Запись модулей выхода
>
> > - Метод: `POST`
> > - url: `http://{IP:PORT}/api/back/output`
> > - Отправляемые данные: `{ Список модулей на запись, Кол-во потоков }`
> > - Получаемые данные: `{ Обновленные показания с модулей }`
> > - Периодичность: `по изменению данных для записи. Мин время = циклу программы 300мс`

## 2. plc_io (опрос модулей)

- Принимает запросы от сервера ангара (алгоритм)
- Опрашивает свои модули и записывает значения в аккумулятор
- Формирует аварийные сообщения о неисправности модулей

> ### Создание микросервиса plc_io состоит из 3 шагов:
>
> #### 1. Привязка микросервиса plc_io к конкретному терминалу (один POS терминал может иметь несколько plc_io):
>
> > Клиент - Форма POS-терминал - перейти на 3 страницу "Микросервис IO" - создать
>
> #### 2. Привязка микросервиса plc_io к складу:
>
> > Склады - Склад - 2 страница "модули" - Таблица Микросервисы - добавляем из выпадающего списка нужный микросервис в склад, который ранее (в п.1) был создан
>
> #### 3. Привязка модулей:
>
> > - Если склад имеет только один микросервис, то все модули склада автоматически подтянутся к микросервису
> > - Если склад имеет несколько микросервисов plc_io, то необходимо добавить каждому микросервису свои модули:
> >     > _Склады - Склад - 2 страница "модули" - Таблица Микросервисы - добавить из выпадающего списка микросервис - Перейти внутрь микросервиса "Таблица модулей" - добавить необходимы модули в микросервис из выпадающего списка_
